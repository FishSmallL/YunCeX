# 1. 401 Client Error: Unauthorized

先配置kaggle api

```bash
# 登入
kaggle auth login
# 配置api
kaggle auth print-access-token
```

将token添加到环境变量中

在Linux中可以放到.bashrc

在win可以放到系统环境变量

注，如果在win中还是报401的错，则用下面这条去临时设置

```
$env:KAGGLE_API_TOKEN="token"
```

如果在Linux也还是报错，则在终端export去设置

`export KAGGLE_API_TOKEN="token"`

# 2. 一些参数的设置

在 `kaggle_knowledge\config.json`

`competitions_per_keyword` 设置每个关键词使用多少个竞赛

`top_leaderboard_users` 每个竞赛，选择前几的用户

`kernels_per_user` 每个用户选择几个相关的kernel来保存

`min_leaderboard_score` 提交的kernel的分数必须要高于这个，否则跳过

`top_skills` 通过匹配算法筛选出的skill，给大模型使用的数量

`min_team_count 筛选竞赛人数，少于则跳过`

# 3. LLM 返回截断

使用大模型去对kernel进行skill提取，如果max_tokens太少，则返回的内容会被截断导致报错

修改为位置 `kaggle_knowledge\kernel_processor\skill_extractor.py`

max_tokens=8192

# 4. 对skill截断处理

kaggle_knowledge\kernel_processor\skill_library.py 的 build_skill_context()

筛选出前 top_skills 个skill后，会对skill的body后1000字符进行截断，避免过长。
